# 我的 PKM 系统规范

> [!summary] 核心原则
> 1. **原子化**：一篇文章只讲一个观点。
> 2. **扁平化**：文件夹仅用于区分状态，不用于区分主题。
> 3. **强链接**：使用 `[[]]` 建立笔记间的有机联系，而非依赖搜索。

## 1. 目录结构

遵循 " 生命周期 " 分类法，而非 " 图书分类法 "。

- **00_Inbox**
	- 收件箱。所有速记、剪藏、未处理的笔记均存放于此。
	- *Action*: 需在 48 小时内处理并移动。
- **10_Notes**
	- 第二大脑核心区。存放所有处理完毕的原子笔记、概念笔记。
	- *Rule*: 不设子文件夹，依靠 Maps 和双链组织。
- **20_Projects**
	- 正在进行的项目、文章输出稿、特定主题的研究。
	- *Example*: `2026年度总结`, `关于Obsidian的博客文章`。
- **30_Maps**
	- 索引地图 (Maps of Content)。用于聚合 `10_Notes` 中的散点。
- **90_Archives**
	- 归档。已完成的项目、过时的资料。
- **99_System**
	- 系统设置。
	- 包含：`Templates` (模板), `Assets` (图片/附件)。

## 2. 命名与创建规范

### 文件命名

- **原则**：文件名即核心概念。
- 正确：`边际效用递减.md`
- 错误：`2026-01-01笔记.md` (日记除外)

### 通用模板 (Template)

所有新笔记必须包含 YAML Frontmatter：

```yaml
---
created: {{date}} {{time}}
tags:
  - concept 
  - seed 
aliases: []
---
```

### 状态流转 (Status)

在 YAML 中标记笔记的成熟度：

- `#status/seed` ：种子。刚记录的想法，未完善。
- `#status/sapling` ：树苗。正在发展中，已建立部分链接。
- `#status/evergreen` ：常青。内容完整，引用丰富，无需大改。

---

## 3. 连接规范 (Links vs Tags)

> [!tip] 黄金法则
> 如果它是一个**实体**或**概念**，用 `[[]]`；如果它是**属性**或**状态**，用 `#`。

| 类型       | 语法        | 适用场景            | 示例                             |
| :------- | :-------- | :-------------- | :----------------------------- |
| **双向链接** | `[[笔记名]]` | 知识关联、引用、上下文补充   | `[[批判性思维]]`, `[[卢曼卡片盒]]`       |
| **标签**   | `#Tag`    | 笔记状态、来源类型、极宽泛分类 | `#status/todo`, `#source/book` |

---

## 4. 资源管理 (Assets)

- **存放位置**：`99_System/Assets`
- **引用格式**：使用相对路径或简短路径。
- **图片命名**：严禁使用 `Pasted image…`。
	- 必须重命名为：`笔记名_时间戳` 或 `语义化名称`。
	- 工具：使用 `Image Converter` 插件自动处理。

---

## 5. 日常工作流 (Workflow)

1. **捕获 (Capture)**：在 `00_Inbox` 中快速记录。
2. **处理 (Process)**：
	- 提炼核心观点 -> 修改文件名。
	- 补充上下文 -> 写入正文。
	- 建立连接 -> 询问自己：" 这与我库里的 `[[什么笔记]]` 有关？"
3. **归档 (File)**：拖入 `10_Notes`。
4. **涌现 (Review)**：当某主题笔记变多，在 `30_Maps` 创建新的 Map 进行汇总。

### 为什么要这样写？

1. **Callout (警告/提示块)**：使用了 `> [!summary]` 和 `> [!tip]`。在 Obsidian 中，这会渲染成带有颜色和图标的方块，非常适合用来突出重点规则。
2. **Wiki Link 预埋**：比如在笔记末尾写了 `[[Obsidian插件推荐清单]]` 等。这些文件可能还不存在，但这符合 " 双链优先 " 的思维——先建立链接，以后点击它就可以直接创建新笔记。
3. **视觉清晰**：利用表格对比 Tag 和 Link 的区别，以后忘了规则，扫一眼这个表格就记住了。
